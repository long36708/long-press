execa ​

Execa 是一个 Node.js 库，可以替代 Node.js 的原生 child_process 模块，用于执行外部命令。 相当于我们在终端输入命令并执行 Execa拥有更好的性能、可靠性和易用性，支持流式传输、输出控制、交互式 shell 等功能，并跨平台兼容 Windows、macOS 和 Linux 等操作系统。 同时，Execa 还支持 Promise API，提供更好的异步控制和异常处理机制。 使用 Execa 可以简化发现和解决常见的子进程处理问题，是 Node.js 开发中非常有用的工具之一。

1. 安装 Execa ​

在项目中使用 Execa 库需要先安装它，可以使用 npm 进行安装：

sh
npm install --save dev execa

先看原生写法： ​

text
const util = require('util');
const exec = util.promisify(require('child_process').exec);

const res = await exec('ls');
// 1. 可以传入cwd参数指定执行的目录
const res = await exec('ls', { cwd: path.resolve(__dirname, '../../test') })
// 2. 另一种指定执行目录的方法是 process.chdir()
process.chdir('../../test');

跟原生的exec的主要区别或优势： ​

  1. Promise支持
  2. 更高的最大缓冲区。100mb而不是200kb。
  3. 按名称执行本地安装的二进制文件。
  4. 在父进程终止时清除派生的进程。
  5. 更具描述性的错误。

execa 相对于内置的 Node.js child_process 模块有几个好处。

首先，execa 公开了一个基于 promise 的 API。这意味着我们可以在代码中使用 async/await，而不需要像使用异步 child_process 模块方法那样创建回调函数。如果我们需要它，还有一个 execaSync 方法可以同步运行命令。

execa 还可以方便地转义并引用我们传递给它的任何命令参数。例如，如果我们传递一个带有空格或引号的字符串，execa 将为我们处理转义。

程序在输出的末尾添加一行新行是很常见的。这有利于命令行的可读性，但在脚本上下文中没有帮助，因此默认情况下，execa 会自动为我们删除这些新行。

如果执行脚本（node）的进程因任何原因死掉，我们可能不希望子进程挂起。execa 会自动为我们清理子进程，确保我们不会出现“僵尸”进程。

使用 execa 的另一个好处是，它支持在 Windows 上使用 Node.js 运行子进程。 它使用了流行的 cross-spawn 包，该包可以解决 Node.js 的已知问题。

execa 方法的参数如下： ​

• command {string}: 执行的命令。例如 ls。 • [args] {string[]}: 命令参数列表，例如 ["-l", "./"]。 • [options] {Object}: 可选选项。包括： • cwd {string}: 运行命令时使用的当前工作目录。默认为 process.cwd()。 • env {Object}: 用于覆盖 process.env 的环境变量对象。 • extendEnv {boolean}: 是否在子进程中扩展 process.env 对象。 • shell {boolean|string}: 是否在操作系统的 shell 中运行命令。默认为 false。可以是一个字符串，该字符串将作为 shell 的路径。 • stdio {string|Array}: 子进程的标准流。默认为 'pipe'。 可以是长度为 3 的数组，分别表示子进程的 stdin、stdout 和 stderr， 也可以是一个字符串，表示将所有流合并到一个流中。 字符串的有效值为 'pipe'、'ignore' 或 'inherit'。 • input {string|Buffer}: 向 stdin 流写入的内容。 • buffer {boolean}: 是否将 stdout 和 stderr 流中的数据存储在内存中。如果为 false，则将流传递给父进程中的流。默认为 true。 • maxBuffer {number}: 允许的 stdout 或 stderr 流中的最大数据量（以字节为单位）。 如果超出了此限制，则子进程将被终止并抛出一个错误。默认为 10MB。 • killSignal {string|number}: 终止子进程使用的信号。 可以是信号名称的字符串（例如 SIGKILL），也可以是信号的数值。默认为 SIGTERM。 • preferLocal {boolean}: 是否优先使用项目中安装的二进制文件而不是全局安装的二进制文件。默认为 true。 • localDir {string}: 用于查找本地安装的二进制文件的目录。 默认为 process.cwd()。 • reject {boolean}: 是否在命令返回非零退出代码时拒绝 Promise。默认为 true。 • timeout {number}: 子进程的超时时间（以毫秒为单位）。如果超时，则进程将被终止并抛出超时错误。默认为 0（无超时时间）。

基本api及用法： ​

execa(file, arguments?, options?) ​

text

// 指定执行目录同样是传入cwd或者process.chdir()；
const { stdout } = await execa('git', ['status'], { cwd: resolve('../demo') });

execa常用的使用场景 ​

execa 是一个用于在 Node.js 中执行外部命令的库，它提供了一种便捷且功能丰富的接口来替代原生的 child_process模块。 以下是几个 execa 常用的使用场景举例：

  1. 运行shell命令并获取输出： 在许多自动化脚本或构建工具中，需要执行诸如 git pull、npm install、webpack 等命令，并可能需要处理它们的输出。使用 execa 可以轻松实现这一点：
javascript
const execa = require('execa');

async function updateRepository() {
	try {
		const { stdout } = await execa('git', ['pull']);
		console.log('Git pull output:', stdout);
	} catch (error) {
		console.error('Error during git pull:', error.stderr);
	}
}

updateRepository();

  1. 捕获命令的详细输出（包括标准输出和错误输出）： 在调试或记录日志时，可能需要同时查看命令的标准输出（stdout）和错误输出（stderr）。execa 允许你合并这两个流：
javascript
const execa = require('execa');

async function runCommandWithFullOutput(command, args) {
	const { stdout, stderr } = await execa(command, args, { stdio: 'pipe' });

	console.log('Standard Output:');
	console.log(stdout);

	console.log('Error Output:');
	console.log(stderr);
}

runCommandWithFullOutput('eslint', ['.']);

javascript
const execa = require('execa');

async function runCommandWithTimeout(command, args, timeoutMs) {
	try {
		await execa(command, args, { timeout: timeoutMs });
		console.log(`Command ${command} completed successfully within the timeout.`);
	} catch (error) {
		if (error.code === 'ETIMEDOUT') {
			console.error(`Command ${command} timed out after ${timeoutMs}ms.`);
		} else {
			console.error(`Error running command ${command}:`, error);
		}
	}
}

runCommandWithTimeout('slow-command', [], 5000); // Timeout after 5 seconds

  1. 交互式命令执行： 对于需要与用户交互的命令，如密码输入或确认操作，execa 支持通过 stdin 传递输入：
javascript
const execa = require('execa');
const prompts = require('prompts');

async function runInteractiveCommand(command, args) {
	const { password } = await prompts({
		type: 'password',
		name: 'password',
		message: 'Enter your password:',
	});

	const child = execa(command, args, { stdin: 'pipe' });
	child.stdin.write(password + '\n');
	child.stdin.end();

	try {
		await child;
		console.log('Command executed successfully.');
	} catch (error) {
		console.error('Error executing command:', error);
	}
}

runInteractiveCommand('sudo', ['apt-get', 'update']);

  1. 跨平台命令执行： execa 自动处理跨平台命令路径问题，确保在不同操作系统上都能正确执行命令：
javascript
const execa = require('execa');

async function openBrowser(url) {
	await execa('open', [url]); // Works on macOS
	// await execa('xdg-open', [url]); // Works on most Linux distributions
	// await execa('start', [url], { shell: true }); // Works on Windows
}

在 Node.js 中使用纯 ES 模块包 ​

Node.js 支持两种不同的模块类型：

CommonJS 模块（CJS）：使用 module.exports 以导出函数和对象，并 require() 将其加载到另一个模块中 ECMAScript 模块（ESM）：使用 export 导出函数和对象，并使用 import 将它们加载到另一个模块中

execa 在其 v6.0.0 版本成为一个纯 ESM 包。这意味着我们必须使用支持 ES 模块的 Node.js 版本才能使用这个包。

对于我们自己的代码，我们可以显示 Node.js 通过在 package.json 中添加 "type": "module"，确保我们项目中的所有模块都是 ES 模块。或者我们可以将单个脚本的文件扩展名设置为 .mjs。

execa最佳实践 ​

使用 execa 执行外部命令时，遵循以下最佳实践可以帮助提高代码的健壮性、可读性和跨平台兼容性：

  1. 使用数组形式传递命令和参数： 将命令和参数分开作为数组元素传递，而不是将它们拼接成一个字符串。**这样可以避免因空格、特殊字符或引号引起的解析问题，尤其在处理用户输入或动态参数时 **：
javascript
// Good
execa('git', ['clone', 'https://github.com/user/repo.git']);

// Bad
execa('git clone https://github.com/user/repo.git');
1
2
3
4

  1. 使用 { shell: true } 选项需谨慎： 当命令需要通过 Shell 解析（例如使用管道、重定向、通配符等 Shell 特性）时，可以设置 { shell: true } 选项。 但要注意，这样做可能会引入 Shell 注入安全风险。 尽量避免在用户输入或不可信数据中使用此选项，或确保对输入进行了充分的验证和转义：
javascript
// 只在必要时使用
execa('echo "$PWD"', { shell: true });
1
2
  1. 处理命令输出： 使用 .then() 或 await 获取命令的 stdout 和 stderr，并适当地处理它们。对于非零退出码，execa 会抛出错误，应通过 try...catch 捕获并处理：
javascript
try {
	const { stdout, stderr } = await execa('ls', ['-l']);
	console.log('Output:', stdout);
	if (stderr) console.warn('Warning:', stderr);
} catch (error) {
	console.error('Error:', error.stderr || error.message);
}

  1. 设置合理的超时： 对于可能长时间运行或阻塞的命令，设置合理的超时限制可以防止进程无限制地挂起。超时后，execa 会抛出一个带有 code: 'ETIMEDOUT' 的错误：
javascript
await execa('long-running-command', { timeout: 60000 }); // 1 minute timeout
1
  1. 捕获并处理所有退出码： 即使命令成功执行，也可能返回非零退出码以表示某种特定状态。通过检查 error.exitCode 来区分不同类型的失败，并据此进行适当处理：
javascript
try {
	await execa('npm', ['install']);
} catch (error) {
	if (error.exitCode === 1) {
		console.error('NPM install failed due to dependency issues.');
	} else {
		console.error('Unknown error occurred:', error);
	}
}

  1. 考虑跨平台兼容性： 不同操作系统可能需要不同的命令或参数。使用条件语句或配置文件来适应不同平台，或者选择跨平台兼容的命令：
javascript
const openCommand = process.platform === 'darwin' ? 'open' : 'xdg-open';
execa(openCommand, ['file.txt']);

  1. 避免在命令中硬编码路径： 使用 path 模块或 execa 的 cwd 选项来处理路径，以确保跨平台兼容性和正确性：
javascript
const filePath = path.join(__dirname, 'file.txt');
execa('cat', [filePath], { cwd: '/some/directory' });
1

  1. 保持更新： execa 会定期发布新版本以修复 bug、添加功能和提高兼容性。确保你的项目使用的是最新版本的 execa。

遵循以上最佳实践，可以确保你的代码在使用 execa 执行外部命令时既高效又稳健，同时考虑到各种边界情况和潜在问题。
